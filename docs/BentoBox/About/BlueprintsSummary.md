# 蓝图（自定义初始岛屿）

当玩家创建新岛屿时，BentoBox 会为他们粘贴一个预先构建的初始岛屿。这些模板称为**蓝图**。默认情况下，每个游戏模式都有一个基本的初始岛屿，但你可以设计自己的。

## 为什么要自定义初始岛屿？

- 给你的服务器独特的外观和风格
- 精确控制玩家开始时拥有的资源
- 创建不同难度的层次（简单初始、困难初始），玩家可以在其间选择
- 为新玩家添加欢迎标志和提示

## 它是如何工作的（简要说明）

蓝图完全在游戏内部创建。基本过程是：

1. 在世界的某处构建你想让玩家开始的岛屿。
2. 使用管理员蓝图命令来选择区域、复制并保存它。
3. 使用蓝图管理器 GUI 将其分配给**束**（一个命名的初始岛屿集合）。
4. 玩家在创建新岛屿时将看到可用的束，并可以选择一个。

你可以对主世界、地狱和末地拥有不同的蓝图，都在同一个束内。

!!! tip
    保持初始岛屿较小。天空方块风格游戏的乐趣在于成长的奋斗。初期资源过多会降低挑战。

## 蓝图管理器

蓝图管理器是一个通过以下方式打开的游戏内 GUI：
```
/[admin_command] blueprint
```

从这里你可以创建新的束、分配蓝图、设置图标、添加描述，以及对特定束（例如 VIP 初始岛屿）要求权限。

## 性能

蓝图粘贴是异步完成的 — 无论蓝图有多大，它都不会冻结你的服务器。如果你在岛屿创建期间注意到延迟，请减少 BentoBox 的 `config.yml` 中的 `paste-speed` 设置。
